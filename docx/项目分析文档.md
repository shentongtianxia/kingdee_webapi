# 项目分析文档 - Kingdee Web API

## 一、项目概述

这是一个 **金蝶K3 Cloud ERP系统的Web API中间件项目**，用于对接金蝶K3 Cloud系统的业务报表数据，通过RESTful API接口对外提供标准化的报表查询服务。

**一句话总结**：将金蝶K3 Cloud的财务报表数据转换为Web API接口，方便其他系统集成调用。

## 三、核心功能

### 3.1 主要职责
1. **接口封装**：将金蝶K3 Cloud的报表功能封装为RESTful API
2. **数据转换**：将金蝶返回的原始数据转换为标准JSON格式
3. **参数映射**：将前端请求参数转换为金蝶SDK所需的格式
4. **API文档**：通过Swagger/Knife4j提供在线接口文档

### 3.2 已实现的报表接口

| 序号 | 报表名称 | 接口路径 | 表单ID |
|------|---------|----------|--------|
| 1 | 科目余额表 | POST /api/k3cloud/reports/account-balance | GL_RPT_AccountBalance |
| 2 | 现金日报表 | POST /api/k3cloud/reports/cash-daily | CN_CashDailyReport |
| 3 | 销售出库明细表 | POST /api/k3cloud/reports/sales-outbound | Sal_OutStockDetailRpt |
| 4 | 生产入库月报表 | POST /api/k3cloud/reports/production-inbound-monthly | PRD_InStockMonthRpt |
| 5 | 销售订单执行明细表 | POST /api/k3cloud/reports/sale-order-detail | SAL_DetailReport |

### 3.3 报表类型覆盖
- **财务会计**：科目余额表、现金日报表
- **销售管理**：销售出库明细表、销售订单执行明细表
- **生产管理**：生产入库月报表

## 五、调用流程

```
客户端请求
    ↓
Controller（接收HTTP请求）
    ↓
Service（构建金蝶API参数）
    ↓
K3CloudApi SDK（调用金蝶云接口）
    ↓
数据解析（将金蝶返回数据转换为Java对象）
    ↓
响应包装（统一格式返回JSON）
    ↓
客户端
```

## 六、API接口规范

### 6.1 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "list": []
  },
  "timestamp": 1703123456789
}
```

### 6.2 典型调用示例
```bash
# 查询科目余额表
curl -X POST http://192.168.1.14:8091/api/k3cloud/reports/account-balance \
  -H "Content-Type: application/json" \
  -d '{
    "acctBookNumber": "001",
    "currency": "1",
    "startYear": "2025",
    "startPeriod": "1",
    "endYear": "2025",
    "endPeriod": "1"
  }'
```

## 七、部署与运行

### 7.1 环境要求
- JDK 1.8+
- Maven 3.6+
- 金蝶K3 Cloud 8.0+ 云环境

### 7.2 运行方式
```bash
# Maven运行
mvn spring-boot:run

# 或打包后运行
```

## 十、快速理解要点

1. **本质**：金蝶K3 Cloud的API网关/中间件
2. **价值**：将复杂的金蝶SDK调用简化为标准HTTP接口
3. **特点**：轻量级、无数据库、纯RESTful API
4. **扩展性**：通过新增Controller和Request/Response模型可快速添加新报表
5. **维护性**：每个报表接口都有独立文档（md目录）和测试数据（txt目录）

## 十一、核心代码片段

### Controller层（ReportController.java）
- 入口：接收HTTP POST请求
- 关键方法：调用K3CloudApi.getSysReportData(formId, jsonData)
- 职责：参数构建、响应映射

### Service层（KingdeeDataSetService）
- 职责：业务逻辑处理
- 实现：数据转换和服务编排

### Model层
- Request类：请求参数封装（如AccountBalanceRequest）
- Data类：数据实体（如FinancialData）
- Response类：响应包装（如Financial）
